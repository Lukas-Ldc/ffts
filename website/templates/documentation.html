The accounts page is the main page of the website. You land on it when you login or when you click on the little home in the top left corner. You can logout by clicking in the top right corner.

On this page you can add, modify or delete an account. You can also view and access your accounts.

In the title of the page, you can see the username of the user who is currently logged in. If you have an administrator account, you will see a little server on the left of the title. You can access the admin menu from it.

These are the differents parameters for an account:

  • Name [Mandatory] (ex : Interactive Brokers) Cannot be changed (do a back-up before trying !).
  • Type [Mandatory] (ex : Broker) In the Django admin menu, go to the standards table to add an account type.
  • Group [Optional] (ex : Long Term Investments).
  • Unit [Mandatory] (ex : USD,EUR) Every transaction should be against an account unit (not mandatory but used in the account page for calculations).
  • GMT [Mandatory] (ex : -5) Modify the hours displayed and also used for the export. In the Django admin menu, go to the standards table and add the local GMT time.
  • Comment [Optional] (ex : Anything).

When you click on the name of an account in the accounts page, you land on the account page. In the title of the page, you can find the account name. The accountS page and the account page are different.

On the account page, you can see the assets of the accounts and the total volume per asset. You can access the transactions, the transfers and also the importation page for the account.

For the assets table (the first one), each asset is linked to only one unit account (unit column). The performance, the profit/loss and the average buy/sold price are calculated with the unit. The assets transactions not against the linked unit are forgotten. The table is first separated between the assets with a balance of zero and then ordered by name.

For the volume table (the second one), the input, output and fee volume are shown for the transactions (TR) and the transfers (TF). The table is ordered by name.

WARNING: the calculations could be wrong, do not relly 100% on it. Don't hesitate to contribute to the project.

From an account page, click on the transactions button to access them.

In the title of the page, there is the account name. You can click on it to go back to the account page.

From this page, you can view, add, modify, delete and export transactions linked to the account.

These are the differents parameters for a transaction:

  • Market [Optional] (ex : NASDAQ)
  • Type [Mandatory] (ex : Trade) In the Django admin menu, go to the standards table to add a transaction type.
  • Date [Mandatory] (ex : 20/10/2020 10:30:05)
  • Input [Mandatory] (ex : USD)
  • Output [Mandatory] (ex : AAPL)
  • Amount In [Mandatory] (ex : 300)
  • Amount Out [Mandatory] (ex : 2)
  • Price [Optional] (ex : 150)
  • Fee [Optional] (ex : 2.5)
  • Fee Type [Optional] (ex : USD)
  • Comment [Optional] (ex : Anything)

You can delete multiple transactions at once.

From an account page, click on the transfers button to access them.

In the title of the page, there is the account name. You can click on it to go back to the account page.

From this page, you can view, add, modify, delete and export transfers linked to the account.

These are the differents parameters for a transfer:

  • Source [Mandatory] (ex : Bank Account)
  • Destination [Mandatory] (ex : Interactive Brokers)
  • Date [Mandatory] (ex : 20/10/2020 10:30:05)
  • Unit [Mandatory] (ex : USD)
  • Amount [Mandatory] (ex : 1000)
  • Fee [Optional] (ex : 1.5)
  • Fee Type [Optional] (ex : USD)
  • Comment [Optional] (ex : Anything)

You can delete multiple transfers at once. For each transfer, one of the accounts must belong to you (user login).

From the account page, click on the importation button to access the importation page.

Select the importation model you want to use and follow the instructions. Don't hesitate to add your own model.

FFTS / Basic CSV

This is the basic importation model for a CSV file generated by an FFTS export or by yourself: comma separated & UTF-8.

You have to choose wether you import transactions or transfers. Fee calculation + fee unit: to be removed.

This is an example for a transaction import:

  • Interactive Brokers,NASDAQ,Trade,2020-01-01 00:00:01-05:00,USD,AAPL,300,2,150,2.5,USD,Anything
  • Interactive Brokers,,Trade,2020-02-22 10:30:00-05:00,USD,GOOG,200,2,100,0,,Anything

This is an example for a transfer import:

  • Bank Account,Interactive Brokers,2020-01-01 00:00:01-05:00,USD,1000,1.5,USD,Anything
  • Bank Account,Interactive Brokers,2020-02-22 10:30:00-05:00,USD,500,,,

The accounts linked to the import have to be added before.

Binance

For the XLSX files, from Excel or Libre Office you can save as CSV. Sometimes the data is altered by the software (ex : the date) and the importation might not work. You also have to remove the first line with the columns names.

These are the differents types of importation:

  • Transactions : Orders > Spot Orders > Trade History > Export (only tested with spot)
  • Crypto deposit : Wallet > Overview > Transaction History > Crypto > Deposit > Export Deposit History
  • Crypto withdrawal : Wallet > Overview > Transaction History > Crypto > Withdrawal > Export Withdrawal History
  • Fiat deposit : Wallet > Overview > Transaction History > Fiat > Withdrawal > Export Withdrawal History
  • Fiat withdrawal : Wallet > Overview > Transaction History > Fiat > Withdrawal > Export Withdrawal History
  • POS/Savings Interests & Small BNB Conversions : Wallet > Overview > Transaction History > Generate all statements
  • POS/Savings Interests Only : Wallet > Overview > Transaction History > Generate all statements (Spot & Savings are enough).
  • BNB Conversions (HTML Extract) : Wallet > Fiat & Spot > Convert Small Balance to BNB > Convert History, then copy the body of the page (source code) in a HTML file and import it (remove line breaks if not working).

Depending on the type of importation, you can have the followings parameters:

  • Transaction Type : the type that must be associated with the imported data (ex : trade, interest, ...).
  • Depost/Withdrawal Account Linked : the account where the transfers are coming/going (manual : the address column has been replaced with the names of the accounts).

For the POS Interests : will only import the interests after redemption/refund detected. Warning : for the same asset, if you have more than one stake which overlap and one of them has been cancelled before the redemption date, the numbers won't be correct.

Degiro

You have to remove the first line with the columns names. These are the differents types of importation:

  • Transactions : Inbox > Transactions > Export > CSV
  • Transfers : Inbox > Account statement > Do not hide cash movements > Export > CSV

For the transfers, you have to make multiple changes in the CSV file. Because the file is not in the same language for everyone, you have to find your deposits, withdrawals and change operations. You can do it automatically with a find an replace option. In column number 6, replace with the following values : "DEPOSIT", "WITHDRAWAL", "CHANGE_IN" and "CHANGE_OUT". Can be improved by sending me the file in order to automate it in your language.

Gate.io

You have to remove the first line with the columns names. These are the differents types of importation:

  • Transactions : Order > Classic account > Spot orders > Trade history > download (only tested with spot)
  • Crypto deposit : Wallet > Funds overview > Deposit > Recent deposits > download
  • Dust (HTML Extract) : Wallet > Funds overview > Spot account > Dust swap > Dust swap history

Depending on the type of importation, you can have the followings parameters:

  • Transaction Type : the type that must be associated with the imported data (ex : trade, interest, ...).
  • Depost/Withdrawal Account Linked : the account where the transfers are coming/going (manual : the address column has been replaced with the names of the accounts).
There are no other types of importation because I don't have any more data to analyse. Don't hesitate to contribute to the project.
Interactive Brokers

There is only one type of importation : "Transactons + Transfers" : Performance & Reports > Activity > Format [CSV] + Language [English]

You also have the following parameters:

  • Transaction Type : the type that must be associated with the imported data (ex : trade, interest, ...).
  • Bank account : assuming that every transfer is coming/going to this account (manual modifications might be needed).
