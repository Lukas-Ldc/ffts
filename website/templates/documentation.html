The accounts page is the main page of the website. You land on it when you login or when you click on the little home in the top left corner. You can logout by clicking in the top right corner.

On this page you can add, modify, delete accounts and also view and access them. You also have the possibility to change your time zone: it will affect the date of the data (on the site & exported).

In the title of the page, you can see the username of the user who is currently logged in. If you have an administrator account, you will see a little server on the left of the title. You can access the Django admin menu from it.

These are the differents parameters for an account:

  • Name [Mandatory] (e.g. Interactive Brokers) Cannot be changed (do a back-up before trying !).
  • Type [Optional] (e.g. Broker) Mandatory: In the Django admin menu, go to the standards table to add an account type.
  • Group [Optional] (e.g. Long Term Investments).
  • Unit [Mandatory] (e.g. USD,EUR) Every transaction should be against an account unit (for correct calculations in the account page). The first unit can sometimes be used as the one by default in some calculations, rank them by importance.
  • UTC [Mandatory] (e.g. Europe/Paris) When you import transactions and transfers without UTC, the date will be saved according to this UTC.
  • Comment [Optional] (e.g. Anything).

When you click on the name of an account in the accounts page, you land on the account page. In the title of the page, you can find the account name. The accountS page and the accountX page are different.

On the account page, you can see a summary of the account's assets. You can access the transactions, the transfers and also the importation page for the account.

The first table gives you a list of all the assets you ever had. The second table gives you a list of all the pairs you traded.

This is when the account units are used. They are automatically put on the right of the pair and the left asset is considered as the one we buy and sell. The fees are linked to the asset at the right.

When there is a pair with two account units, the one at the right will be the first one found in the unit field of the account.

For a string that is not mandatory, whether it is an account, a transaction or a transfer, if you enter "0", it will clean the string to ''.

From an account page, click on the transactions button to access them.

In the title of the page, there is the account name. You can click on it to go back to the account page.

From this page, you can view, add, modify, delete and export transactions linked to the account.

These are the differents parameters for a transaction:

  • Market [Optional] (e.g. NASDAQ)
  • Type [Optional] (e.g. Trade) Optional: In the Django admin menu, go to the standards table to add a transaction type.
  • Date [Mandatory] (e.g. 20/10/2020 10:30:05)
  • UTC [Mandatory] (e.g. Europe/Paris)
  • Input [Mandatory] (e.g. USD)
  • Output [Mandatory] (e.g. AAPL)
  • Amount In [Mandatory] (e.g. 300)
  • Amount Out [Mandatory] (e.g. 2)
  • Price [Optional] (e.g. 150)
  • Fee [Optional] (e.g. 2.5)
  • Fee Unit [Optional] (e.g. USD)
  • Comment [Optional] (e.g. Anything)

You can modify or delete multiple transactions at once (ID1,ID2,ID3,...).

From an account page, click on the transfers button to access them.

In the title of the page, there is the account name. You can click on it to go back to the account page.

From this page, you can view, add, modify, delete and export transfers linked to the account.

These are the differents parameters for a transfer:

  • Source [Mandatory] (e.g. Bank Account)
  • Destination [Mandatory] (e.g. Interactive Brokers)
  • Date [Mandatory] (e.g. 20/10/2020 10:30:05)
  • UTC [Mandatory] (e.g. Europe/Paris)
  • Unit [Mandatory] (e.g. USD)
  • Amount [Mandatory] (e.g. 1000)
  • Fee [Optional] (e.g. 1.5)
  • Fee Unit [Optional] (e.g. USD)
  • Comment [Optional] (e.g. Anything)

You can modify or delete multiple transfers at once (ID1,ID2,ID3,...). For each transfer, one of the accounts must belong to you (user loged in).

From the account page, click on the importation button to access the importation page.

Select the importation model you want to use and follow the instructions. Don't hesitate to add your own model.

When importing a file, you choose an UTC. For each line, if the date is aware of the UTC (e.g. "2022-11-15 03:00:00+01:00"), it will prevail, else the selected UTC is used.

FFTS / Basic CSV

This is the basic importation model for a CSV file generated by an FFTS export or by yourself: comma separated & UTF-8. You have to choose wether you import transactions or transfers. Protected against duplicates.

This is an example for a transaction import:

  • user_Interactive Brokers,NASDAQ,Trade,2020-01-01 00:00:01-05:00,USD,AAPL,300,2,150,2.5,USD,Anything
  • user_Interactive Brokers,,Trade,2020-02-22 10:30:00-05:00,USD,GOOG,200,2,100,0,,Anything

This is an example for a transfer import:

  • user_Bank Account,user_Interactive Brokers,2020-01-01 00:00:01-05:00,USD,1000,1.5,USD,Anything
  • user_Bank Account,user_Interactive Brokers,2020-02-22 10:30:00-05:00,USD,500,,,

The accounts linked to the import have to be added before. For the transfer import, you can precise the name (user_name) of the account that will be replaced by the actual account. If you wish to make more modification, use a find and replace tool.

Binance

For the XLSX files, from Excel or Libre Office you can save as CSV: sometimes the data is altered by the software (like the date) and the importation might not work. Protected against duplicates. The export button is on the top right corner of the website.

These are the differents types of importation:

  • Transactions : Orders > Spot Orders > Trade History > Export (only tested with spot)
  • Crypto deposit : Orders > Transaction History > Crypto > Type: Deposit > Export Deposit History
  • Crypto withdrawal : Orders > Transaction History > Crypto > Type: Withdrawal > Export Withdrawal History
  • Crypto Buy: Orders > Fiat Order > Buy > Export
  • Fiat deposit : Orders > Fiat Order > Fiat Deposit > Export
  • Fiat withdrawal : Orders > Fiat Order > Fiat Withdrawal > Export
  • POS/Savings Interests & Small BNB Conversions : Orders > Transaction History > Export Transaction Records
  • POS/Savings Interests Only : Orders > Transaction History > Export Transaction Records
  • BNB Conversions (HTML Extract) : Wallet > Transaction History > BNB Convert (open the conversions you want to import), then copy the body of the page (source code) in an HTML file and import it.
  • Binance C2C Transfer : Orders > Transaction History > Export Transaction Records (read below)

Depending on the type of importation, you can have the followings parameters:

  • Transaction Type : the type that must be associated with the imported data (e.g. trade, interest, ...).
  • Depost/Withdrawal Account Linked : the account where the transfers are coming/going (manual : the address column has been replaced with the names of the accounts).

For the locked subscription (POS Interests) : will only import the interests after subscription + redemption detected in the same CSV. If you have more than one stake for the same asset which overlap and one of them has been cancelled, the numbers won't be correct. If you have for the same asset a flexible and a locked subscription, the numbers won't be correct.

When importing transfers between accounts (C2C), you can use "Manual (Crypto D/W only)" to automatically link the transfers with the right accounts. In order to work, you have to replace inside the CSV file the column containing the address with the unique ID of the FFTS account.

Degiro

Protected against duplicates. These are the differents types of importation:

  • Transactions : Inbox > Transactions > Export > CSV
  • Transfers : Inbox > Account statement > Do not hide cash movements > Export > CSV

For the transfers, you have to make multiple changes in the CSV file. Because the file is not in the same language for everyone, you have to find your deposits, withdrawals and change operations in order to rename a column so it can be detected. You can do it automatically with a find an replace option. In column number 6, replace with the following values : "DEPOSIT", "WITHDRAWAL", "CHANGE_IN" and "CHANGE_OUT". Supported languages (modifications not needed) : French. Can be improved by sending me the file in order to automate it in your language.

EstateGuru

There is one type of importation (automatic): Account statement > Details > Download detailed rapport (CSV)

The only parameter is your bank account used to make deposits and withdrawals to your EstateGuru account. Protected against duplicates.

Gate.io

Protected against duplicates. These are the differents types of importation:

  • Transactions : Order > Classic account > Spot orders > Trade history > download (only tested with spot)
  • Crypto deposit : Wallet > Funds overview > Deposit > Recent deposits > download
  • Dust (HTML Extract) : Wallet > Funds overview > Spot account > Dust swap > Dust swap history

Depending on the type of importation, you can have the followings parameters:

  • Transaction Type : the type that must be associated with the imported data (e.g. trade, interest, ...).
  • Depost/Withdrawal Account Linked : the account where the transfers are coming/going (manual : the address column has been replaced with the names of the accounts).
There are no other types of importation because I don't have any more data to analyse. Don't hesitate to contribute to the project.
Interactive Brokers

Protected against duplicates. There is only one type of importation : "Transactons + Transfers" : Performance & Reports > Activity > Format [CSV] + Language [English]

You also have the following parameters:

  • Transaction Type : the type that must be associated with the imported data (e.g. trade, interest, ...).
  • Bank account : assuming that every transfer is coming/going to this account (tagged "Electronic Fund Transfer").
  • Internal account (IB account) : assuming that every transfer is coming/going to this account (tagged "Internal Transfer").
